# Keycloak Upgrade (15 to 26)

This release upgrades Keycloak from version 15 to version 26 across backend
services. A key reason for this work is to support SDX enhancements while
improving security and compatibility with current authentication standards.

## Backend Updates

- **Authentication integration updates** - Updated backend authentication flows
  and service integrations to support Keycloak 26 for SDX-related work.
- **Compatibility hardening** - Adjusted backend behavior to align with
  Keycloak 26 token handling and modern identity provider expectations.

## API Services Portal (3.0.6 to 3.0.8)

The API Services Portal was upgraded from version 3.0.6 to 3.0.8 to support
Keycloak 26 compatibility and include additional reliability updates.

### Changes

- **Keycloak 26 compatibility** - Updated portal authentication and related
  logic to support Keycloak 26.
- **Notifications coverage** - Added end-to-end tests for email notifications.
- **Cypress reliability fixes** - Included test stability updates in 3.0.8.

## `gwa-api` Updates (latest two releases)

## Additional `gwa-cli` Login Prompt

When logging into the `gwa-cli`, there is now an additional prompt:

Do you grant these access privileges?
- gwa-cli

This behavior is expected in Keycloak's device authorization flow: consent is
intended to be shown on every login (including when a user has already
consented), in line with RFC 8628 (OAuth 2.0 Device Authorization Grant) and
its explicit user-interaction requirement. In short, this is by design rather
than a misconfiguration. See
[RFC 8628: OAuth 2.0 Device Authorization Grant](https://www.rfc-editor.org/rfc/rfc8628)
for details.
