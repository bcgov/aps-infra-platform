# Keycloak Update

This release updates the Keycloak client used by `gwa-cli` to require PKCE 
(Proof Key for Code Exchange) with S256 (SHA-256) for the OAuth 2.0 Device 
Flow.

This is a breaking change for older versions of `gwa-cli`, because they do not 
support PKCE with S256. Users must upgrade to `gwa-cli` version `3.1.0` or 
later.

The purpose of this change is to improve security and align authentication 
behaviour with current OAuth 2.0 best practices.

# `gwa-cli` Update (3.1.0)

The `gwa-cli` command line application has been updated to support PKCE (Proof 
Key for Code Exchange) with S256 (SHA-256).

Previous versions of `gwa-cli` did not implement PKCE. The updated version now 
defaults to PKCE with S256, which improves the security of the OAuth 2.0 
Device Flow used during authentication.

No changes are required to how users run or interact with the CLI. PKCE with 
S256 is enabled by default.

Users should ensure they are using the latest version of `gwa-cli`. 
Instructions for obtaining the latest version are available in the 
[gwa-cli installation guide](https://developer.gov.bc.ca/docs/default/component/aps-infra-platform-docs/how-to/gwa-install/).
